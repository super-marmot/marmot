# Adequacy audit

Lazy artifacts that must fail:

1. A visual chip with no functionality: the verifier requires action
   registration, a local execution hook, typed-card metadata, and a focused
   test.
2. A model-backed privacy claim: the verifier requires the local execution
   branch and rejects the absence of representative redaction markers.
3. An emoji or generic icon: the verifier requires a semantic `privacy` icon
   mapped to a native shield symbol in the shared icon primitive.
4. Overclaiming complete privacy protection: the rubric limits this milestone
   to common email, phone, URL, card, and SSN-like values; names, addresses,
   and arbitrary identifiers remain out of scope.
