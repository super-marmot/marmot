# Verifier adequacy audit

Lazy artifacts that must fail:

1. A filename-only attachment path: the verifier requires `File.text()` and
   the awaited completion-history builder.
2. Unbounded file injection: the verifier requires a named context cap and a
   focused truncation test.
3. A UI claim that every attachment is understood: the verifier requires an
   explicit unsupported-media boundary and a capability label in the chip.
4. A composer that passes the native press event into the string override:
   the verifier requires an explicit zero-argument `send()` binding.

Final acceptance requires a real Android attachment picked from the system
picker, visible capability status, and a locally grounded answer from the
loaded model.
