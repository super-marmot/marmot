# Verifier adequacy audit

The verifier is frozen before implementation and combines source contracts
with a focused behavior test.

Lazy artifacts that must fail:

1. A drawer can be visually softened while still displaying the internal
   `No visible answer yet` status; the preview and conditional-render checks
   block that regression.
2. A restyle can improve contrast while shrinking tap targets; the outlined
   action contract still requires a 44pt new-chat control.
3. A source-only change can claim to hide reasoning while the preview helper
   returns the old placeholder; the focused Jest test executes the behavior.

The final acceptance bar is completed by a fresh Android emulator screenshot of
the drawer with a short history and a search interaction, plus the repository
test, typecheck, export, and corpus checks.
