# Flight mode verifier rubric

Flight Mode must be a deliberate, user-invoked offline surface. It should
offer several engaging activities through the already-loaded local model,
bound context and output for mobile memory/battery, disable web access, and
make stopping generation explicit. Leaving the screen must not imply an
always-on background agent.

The companion activity is a private session primitive in this milestone; it
does not silently write memories, schedule notifications, or run in the
background. Persistent milestones require a later explicit consent flow.
