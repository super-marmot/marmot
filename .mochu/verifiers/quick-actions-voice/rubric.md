# Quick actions voice verifier rubric

Quick actions should accept spoken commands without forcing the user into the
full Voice screen. The mic is a text-entry affordance: request permission,
show interim recognition, append the final transcript to any typed command,
and leave the user in control of which action runs.

The action catalog should cover the common moments after sharing or pasting
content: understand, write, plan, and protect. Grouping is required so a
larger catalog remains scannable on a phone.
