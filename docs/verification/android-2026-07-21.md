# Android UI verification — 2026-07-21

Target: `marmot` AVD, Pixel 7 profile, Android 35, x86_64, 1536 MB RAM.

Build: development APK installed with `npx expo run:android --no-bundler` and
Metro served through an ADB reverse tunnel on port 8081.

Verified scenarios:

- Chat list renders clean Markdown previews without Markdown delimiters or
  leaked reasoning scaffolding.
- The header menu opens a left navigation drawer with grouped/searchable chat
  history, New chat, Quick actions, Model library, and Settings.
- Short-history drawer rows omit empty assistant output instead of showing
  internal answer-status language; the outlined New chat action, search field,
  footer, scrim, and shadow remain visually balanced on the phone.
- A chat opens with a reusable model dropdown showing model details and the
  selected model state.
- Composer actions use professional icons with accessible touch targets; no
  emoji controls are present.
- Drawer and selector open/close transitions render correctly on the emulator.
- Model library shows a RAM-derived recommended starting point and explains
  the fit rationale without promising a speed result.
- A ready recommended model exposes `Try offline demo`; Chat shows the
  local-only proof card, loads the real model, and returns `Paris` for the
  supplied capital-of-France prompt. The demo disables long reasoning and
  caps its output so the first proof is fast on the phone.
- Quick actions turn shared text into typed result cards: generated summaries
  are labeled as local previews, draft replies are marked `Preview only · Not
  sent`, and saving requires a second explicit approval tap before the local
  document write.
- Quick actions turn shared text into a calendar preview with a resolved local
  time. The Pixel 7 flow accepted calendar permission, created a local Marmot
  calendar when no writable calendar existed, displayed `Added to calendar`, and
  then displayed `Removed from calendar` after `Undo event`. The input
  `Team sync tomorrow at 10 AM` rendered as `7/22/2026 · 10:00 AM–11:00 AM`.
- The attachment picker selected `marmot-attachment-sample.txt`, the composer
  labeled it `Included locally`, and the chat sent `What is the private code
  word`. The local Qwen3.5 response was `snowdrop`; the fixture also contained
  an instruction-like line, which was treated as reference data rather than an
  instruction. The image path then downloaded the paired SmolVLM 256M model and
  projector, labeled the screenshot `Ready for vision`, and returned `Page
  showing options.` locally at 58.5 tok/s. PDF/audio remain separate open
  milestones.
- Quick actions displayed the professional `Summarize`, `Action items`, and
  `PII eraser` chips. The PII eraser ran without loading a model and returned a
  copyable `PII removed` preview with common email, phone, and URL markers.
- The history drawer displayed `Flight mode`; the screen showed the Offline
  companion proof, five bounded activities, the no-network/no-location/no-
  background contract, local model status, and an explicit Stop action. A real
  Travel trivia generation completed in the result card without network tools.
