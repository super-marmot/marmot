# Security policy

Marmot is local-first: screenshots, messages, model files, and action drafts
should stay on the phone unless the user explicitly shares them. Please do not
post private attachments, API keys, keystores, signing credentials, or personal
messages in a public issue.

## Reporting a vulnerability

For a security issue in the app, release pipeline, share handling, or privacy
boundary, contact the repository owner privately through the GitHub security
advisory flow at <https://github.com/stancsz/marmot/security/advisories/new>.
If that flow is unavailable, open a minimal issue asking for a private contact
without including exploit details.

Include the affected version, platform, reproduction steps, and whether any
data left the device. We will acknowledge reports when practical and will
coordinate disclosure after a fix or mitigation is available.
