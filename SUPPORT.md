# Support

Start with the [README](README.md), [contributor guide](CONTRIBUTING.md), and
[reproducible benchmark protocol](docs/BENCHMARKS.md).

For a reproducible bug, use the bug-report issue template and include the app
version, device tier, model, steps, expected result, actual result, and a
sanitized log or screenshot. Do not attach private messages, receipts, API
keys, signing material, or unredacted personal data.

Questions and product discussion belong in GitHub Discussions. Use an issue
when there is a concrete fix or reproducible failure.
